# Финальное исправление RuntimeError: Event loop is closed

## Проблема

Application создается в основном потоке, а затем пытается инициализироваться в другом потоке с другим event loop. Это вызывает конфликт.

## Решение

Упростить архитектуру - не инициализировать Application вручную, а использовать его встроенные механизмы. Application должен запускаться и управлять своим event loop самостоятельно.

## Что нужно сделать

1. Убрать ручную инициализацию `application.initialize()`
2. Использовать `application.start()` который сам создаст event loop
3. Добавлять обновления в очередь через `application.update_queue.put_nowait()`

## Альтернативное решение

Вообще не запускать Application отдельно, а просто использовать его очередь для обновлений. Но для этого нужен работающий Application.
