# Диагностика проблемы с платежами

## Проблема
Бот показывает "Оплата отменена" даже когда платеж успешен и статус записан в базу данных как 'succeeded'.

## По логам видно:
1. ✅ Webhook от YooKassa получен: `payment.succeeded` для платежа `30c55e64-000f-5000-b000-1072b037d9b9`
2. ✅ Статус платежа обновлен в базе: `status=succeeded, user_id=724281972`
3. ✅ Пользователь помечен как оплативший
4. ✅ Платеж успешно обработан через webhook

## Что проверяем:

1. **Статус записывается в БД?** 
   - ДА, видно в логах: `💾 Статус платежа обновлен: payment_id=..., status=succeeded`

2. **Что происходит при возврате пользователя?**
   - Пользователь возвращается через `/start payment_cancel`
   - Бот должен проверить статус последнего платежа
   - Если статус 'succeeded', обработать платеж и НЕ показывать отмену

3. **Почему может показываться "Оплата отменена"?**
   - Пользователь возвращается ДО того, как webhook обработался (race condition)
   - Проблема с чтением статуса из БД
   - Проблема с обработкой платежа

## Решение:
Добавлена проверка статуса в начале обработки `payment_cancel`:
- Если статус уже 'succeeded', сразу обрабатываем платеж
- Улучшено логирование для диагностики
- Добавлена обработка случая, когда платеж успешен, но не обработан автоматически
