# Диагностика проблем с оплатой 💳

## Почему оплата может быть отменена?

### 1. Причины со стороны пользователя

#### Недостаточно средств
- **Причина:** На карте недостаточно средств для оплаты
- **Решение:** Попросите пользователя проверить баланс карты или использовать другую карту

#### Неверные данные карты
- **Причина:** Неправильно введен номер карты, срок действия или CVV/CVC код
- **Решение:** Попросите пользователя внимательно проверить данные карты

#### Истекший срок действия карты
- **Причина:** Карта просрочена
- **Решение:** Пользователь должен использовать действующую карту

#### Блокировка карты банком
- **Причина:** Банк заблокировал карту по различным причинам (подозрение в мошенничестве, превышение лимитов и т.д.)
- **Решение:** Пользователь должен обратиться в свой банк

### 2. Причины со стороны банка

#### Ограничения банка
- **Причина:** Банк установил лимиты на сумму или количество транзакций
- **Решение:** Пользователь должен связаться с банком для снятия ограничений

#### Банк-эмитент недоступен
- **Причина:** Временные проблемы с банком-эмитентом карты
- **Решение:** Попросите пользователя попробовать позже

#### Ошибка 3D Secure
- **Причина:** Проблемы с аутентификацией 3D Secure
- **Решение:** Пользователь должен повторить попытку оплаты

### 3. Технические причины

#### Проблемы с интернет-соединением
- **Причина:** Нестабильное соединение во время оплаты
- **Решение:** Попросите пользователя проверить интернет и попробовать снова

#### Истекло время на оплату
- **Причина:** Пользователь не завершил оплату в течение отведенного времени
- **Решение:** Пользователь должен создать новую ссылку на оплату

#### Проблемы с настройками YooKassa
- **Причина:** Неправильно настроены параметры магазина в YooKassa
- **Решение:** Проверьте настройки в личном кабинете YooKassa

## Как диагностировать проблему?

### 1. Проверьте логи бота

Ищите записи с тегами:
- `payment_canceled` - отмена платежа
- `payment_error` - ошибки при создании платежа
- `yookassa` - все события связанные с YooKassa

### 2. Проверьте статус платежа в YooKassa

1. Войдите в личный кабинет YooKassa
2. Перейдите в раздел "Платежи"
3. Найдите платеж по ID (из логов бота)
4. Проверьте статус и причину отмены (если есть)

### 3. Проверьте настройки webhook

Webhook необходим для получения уведомлений о статусе платежей от YooKassa.

**Проверьте переменную окружения:**
```bash
YOOKASSA_WEBHOOK_URL=https://ваш-домен.com/webhook/yookassa
```

**Важно:**
- URL должен быть доступен из интернета (не localhost)
- URL должен использовать HTTPS
- URL должен быть зарегистрирован в настройках магазина YooKassa

**Как зарегистрировать webhook в YooKassa:**
1. Войдите в личный кабинет YooKassa
2. Перейдите в "Настройки" → "Уведомления"
3. Добавьте URL вашего webhook
4. Выберите события: `payment.succeeded` и `payment.canceled`

### 4. Проверьте переменные окружения

Убедитесь, что все необходимые переменные установлены:

```bash
YOOKASSA_SHOP_ID=ваш_shop_id
YOOKASSA_SECRET_KEY=ваш_secret_key
YOOKASSA_WEBHOOK_URL=https://ваш-домен.com/webhook/yookassa
PAYMENT_SUCCESS_URL=https://t.me/ваш_бот?start=payment_success
PAYMENT_RETURN_URL=https://t.me/ваш_бот?start=payment_cancel
TELEGRAM_BOT_USERNAME=ваш_бот
```

### 5. Проверьте доступность webhook

Выполните запрос к вашему webhook endpoint:

```bash
curl -X POST https://ваш-домен.com/webhook/yookassa \
  -H "Content-Type: application/json" \
  -d '{"test": "data"}'
```

Должен вернуться ответ (даже если это ошибка валидации).

## Улучшения в коде

После последних изменений бот теперь:

1. **Извлекает причину отмены** из webhook YooKassa
2. **Показывает пользователю понятное сообщение** о причине отмены
3. **Логирует детальную информацию** об отмене для диагностики
4. **Проверяет статус платежа** при возврате пользователя в бот

## Частые причины отмены платежей

Согласно статистике YooKassa, наиболее частые причины:

1. **insufficient_funds** (30-40%) - Недостаточно средств
2. **call_issuer** (20-25%) - Банк отклонил платеж
3. **expired_on_confirmation** (10-15%) - Истекло время на оплату
4. **invalid_csc** (5-10%) - Неверный CVV/CVC код
5. **3d_secure_failed** (5-10%) - Ошибка 3D Secure

## Что делать, если проблема повторяется?

1. **Попросите пользователя связаться с банком** - часто банк может дать точную причину отклонения
2. **Проверьте настройки магазина в YooKassa** - убедитесь, что все способы оплаты включены
3. **Проверьте логи бота** - там может быть дополнительная информация
4. **Свяжитесь с поддержкой YooKassa** - они могут помочь диагностировать проблему
