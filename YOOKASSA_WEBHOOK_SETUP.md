# Настройка и проверка Webhook YooKassa 🔔

## Что такое Webhook?

Webhook — это механизм, который позволяет YooKassa автоматически отправлять уведомления о статусе платежей на ваш сервер в реальном времени. Это более надежный способ, чем периодическая проверка статусов.

## Преимущества использования Webhook

✅ **Мгновенные уведомления** — вы узнаете о статусе платежа сразу  
✅ **Надежность** — не нужно периодически проверять статусы  
✅ **Автоматическая обработка** — платежи обрабатываются автоматически  
✅ **Меньше нагрузки на API** — не нужно делать запросы каждые 2 минуты

## Шаг 1: Определение URL вашего webhook

### Для локальной разработки (не рекомендуется для продакшена)

Если вы тестируете локально, используйте ngrok или аналогичный сервис:

```bash
# Установите ngrok
brew install ngrok  # macOS
# или скачайте с https://ngrok.com/

# Запустите туннель
ngrok http 8080

# Скопируйте HTTPS URL (например: https://abc123.ngrok.io)
```

### Для продакшена (Railway, Render, VPS и т.д.)

Ваш webhook URL должен быть:
```
https://ваш-домен.com/webhook/yookassa
```

**Примеры:**
- Railway: `https://your-app.railway.app/webhook/yookassa`
- Render: `https://your-app.onrender.com/webhook/yookassa`
- VPS: `https://your-domain.com/webhook/yookassa`

**Важно:**
- URL должен быть доступен из интернета (не localhost)
- URL должен использовать HTTPS (обязательно!)
- URL должен быть зарегистрирован в YooKassa

## Шаг 2: Настройка переменных окружения

### Локально (файл `.env`)

```bash
# Webhook URL для YooKassa
YOOKASSA_WEBHOOK_URL=https://ваш-домен.com/webhook/yookassa

# Порт для webhook сервера (опционально, по умолчанию 8080)
WEBHOOK_PORT=8080
```

### Railway.app

1. Откройте ваш проект на https://railway.app
2. Перейдите в раздел **"Variables"**
3. Добавьте переменную:
   - **Key:** `YOOKASSA_WEBHOOK_URL`
   - **Value:** `https://your-app.railway.app/webhook/yookassa`
   - **Key:** `WEBHOOK_PORT` (опционально)
   - **Value:** `8080`

### Render.com

1. Откройте ваш проект на https://render.com
2. Перейдите в раздел **"Environment"**
3. В секции **"Environment Variables"** добавьте:
   - **Key:** `YOOKASSA_WEBHOOK_URL`
   - **Value:** `https://your-app.onrender.com/webhook/yookassa`

### VPS / Другие платформы

Добавьте переменные окружения через:
- Systemd service файл
- Docker environment
- Или другие методы вашей платформы

## Шаг 3: Регистрация Webhook в YooKassa

### Через личный кабинет YooKassa

1. **Войдите в личный кабинет YooKassa:**
   - Перейдите на https://yookassa.ru/
   - Войдите в свой аккаунт

2. **Перейдите в настройки магазина:**
   - В меню выберите **"Настройки"** → **"Уведомления"**
   - Или перейдите напрямую: https://yookassa.ru/my/merchant/integration/api/notifications

3. **Добавьте URL webhook:**
   - Нажмите **"Добавить URL"** или **"Настроить"**
   - Введите ваш webhook URL: `https://ваш-домен.com/webhook/yookassa`
   - Выберите события для уведомлений:
     - ✅ `payment.succeeded` — успешная оплата
     - ✅ `payment.canceled` — отмена платежа
   - Нажмите **"Сохранить"**

### Через API YooKassa (альтернативный способ)

Вы также можете зарегистрировать webhook через API:

```bash
curl -X POST https://api.yookassa.ru/v3/webhooks \
  -u "YOUR_SHOP_ID:YOUR_SECRET_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "event": "payment.succeeded",
    "url": "https://ваш-домен.com/webhook/yookassa"
  }'
```

## Шаг 4: Проверка работы Webhook

### 1. Проверка доступности endpoint

```bash
# Проверьте, что endpoint доступен
curl -X POST https://ваш-домен.com/webhook/yookassa \
  -H "Content-Type: application/json" \
  -d '{"test": "data"}'
```

Должен вернуться ответ (даже если это ошибка валидации).

### 2. Проверка через скрипт

Используйте скрипт `check_webhook.py`:

```bash
python check_webhook.py
```

### 3. Проверка в логах бота

После настройки webhook, при запуске бота вы должны увидеть:

```
🌐 Запуск webhook сервера на 0.0.0.0:8080
```

### 4. Тестовая оплата

1. Создайте тестовый платеж через бота
2. Проверьте логи бота — должны появиться сообщения:
   ```
   🔔 Получен webhook от ЮKassa: event=payment.succeeded, payment_id=...
   ✅ Платеж ... успешно обработан через webhook для пользователя ...
   ```

## Шаг 5: Диагностика проблем

### Проблема: Webhook не запускается

**Симптомы:**
- В логах нет сообщения "🌐 Запуск webhook сервера"
- Появляется предупреждение "⚠️ YOOKASSA_WEBHOOK_URL не установлен"

**Решение:**
1. Проверьте, что `YOOKASSA_WEBHOOK_URL` установлен в переменных окружения
2. Убедитесь, что URL начинается с `https://`
3. Перезапустите бота

### Проблема: Webhook не получает уведомления

**Симптомы:**
- Webhook запущен, но уведомления не приходят
- Платежи остаются в статусе `pending`

**Решение:**
1. Проверьте, что webhook зарегистрирован в личном кабинете YooKassa
2. Убедитесь, что URL доступен из интернета:
   ```bash
   curl -I https://ваш-домен.com/webhook/yookassa
   ```
3. Проверьте, что URL использует HTTPS (обязательно!)
4. Проверьте логи бота на наличие ошибок

### Проблема: Ошибки при обработке webhook

**Симптомы:**
- В логах появляются ошибки при обработке webhook
- Платежи не обрабатываются

**Решение:**
1. Проверьте логи бота на наличие ошибок
2. Убедитесь, что база данных доступна
3. Проверьте, что все необходимые переменные окружения установлены

### Проблема: Webhook получает уведомления, но платежи не обрабатываются

**Симптомы:**
- В логах есть сообщения о получении webhook
- Но платежи не обновляются в базе данных

**Решение:**
1. Проверьте логи на наличие ошибок обработки
2. Убедитесь, что `user_id` присутствует в metadata платежа
3. Проверьте подключение к базе данных

## Проверка текущей настройки

Используйте скрипт для проверки:

```bash
python check_webhook_setup.py
```

Скрипт проверит:
- ✅ Наличие переменной `YOOKASSA_WEBHOOK_URL`
- ✅ Формат URL (должен быть HTTPS)
- ✅ Доступность endpoint
- ✅ Регистрацию webhook в YooKassa (если возможно)

## Безопасность

⚠️ **Важные моменты:**

1. **HTTPS обязателен** — YooKassa отправляет уведомления только на HTTPS URL
2. **Валидация подписи** — в будущем можно добавить проверку подписи запросов от YooKassa
3. **Ограничение доступа** — рассмотрите возможность добавления базовой аутентификации

---

## Альтернатива: Периодическая проверка

Если webhook не настроен, бот будет использовать периодическую проверку статусов платежей (каждые 2 минуты). Это менее эффективно, но работает.

**Когда использовать периодическую проверку:**
- Локальная разработка
- Временные проблемы с webhook
- Тестирование без публичного URL
