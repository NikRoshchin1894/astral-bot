# 🔄 Обновление архитектуры бота

## ⚠️ ВАЖНО: Требуется обновление на сервере!

Бот был переведен на упрощенную архитектуру:
- **Один event loop** (через `asyncio.run()`)
- **Один HTTP-сервер** (aiohttp вместо Flask/Werkzeug)
- **Убраны все потоки** и множественные event loops

## 📋 Шаги для обновления:

### 1. Обновить код на сервере

```bash
# На сервере выполните:
git pull origin main
```

### 2. Убедиться, что установлены зависимости

```bash
pip install -r requirements.txt
```

**Важно:** Убедитесь, что в `requirements.txt` есть:
- `aiohttp` (для нового webhook сервера)
- `python-telegram-bot` (версия 20.x)

### 3. Перезапустить бота

**Для Railway/Render:**
- Просто сделайте redeploy через интерфейс
- Или перезапустите сервис

**Для VPS/сервера:**
```bash
# Остановить старый процесс
pkill -f "python.*bot.py"

# Запустить новый
python3 bot.py
```

### 4. Проверить логи

После перезапуска в логах должно быть:
```
🚀 Запуск бота в упрощенной архитектуре (один event loop, aiohttp)
✅ Webhook server (aiohttp) запущен на порту 8080
```

**НЕ должно быть:**
- ❌ "Flask сервер получил сигнал остановки"
- ❌ "Werkzeug server остановлен"
- ❌ "Запуск webhook сервера на..."

## 🔍 Проверка работы

1. Проверьте healthcheck: `curl https://your-domain.com/health`
2. Проверьте, что кнопки бота работают
3. Проверьте логи на отсутствие ошибок "Event loop is closed"

## ❌ Если видите старые логи (Flask/Werkzeug)

Это означает, что на сервере все еще работает старая версия кода:
1. Убедитесь, что выполнили `git pull`
2. Убедитесь, что перезапустили бота
3. Проверьте, что используется правильный файл `bot.py`

## ✅ После обновления

Архитектура теперь:
- ✅ Один event loop - нет конфликтов
- ✅ Один HTTP-сервер - проще управление
- ✅ Нет потоков - нет проблем с синхронизацией
- ✅ Правильная обработка остановки - нет ошибок "different loop"
