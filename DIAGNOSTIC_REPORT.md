# Диагностика проблемы генерации натальной карты для @SberSBI

## Возможные причины проблемы

### 1. Не заполнен профиль полностью ✅
**Проверка:** Генерация запускается только если заполнены все поля:
- Имя (birth_name)
- Дата рождения (birth_date)
- Время рождения (birth_time)
- Место рождения (birth_place)

**Логирование:** Событие `natal_chart_request_no_profile` логируется, если профиль неполный.

**Решение:** Пользователь должен заполнить все поля через "📋 Данные о рождении" → "✏️ Редактировать данные".

---

### 2. Зависшая предыдущая генерация ⏳
**Проверка:** Код проверяет, есть ли незавершенная генерация (стартовала более 10 минут назад).

**Логирование:** Событие `natal_chart_error` с типом `StuckGeneration`.

**Решение:** 
- Проверить события в БД: найти `natal_chart_generation_start` без последующих `natal_chart_success` или `natal_chart_error`
- Если генерация зависла, можно попросить пользователя попробовать снова

---

### 3. Отсутствует OPENAI_API_KEY 🔑
**Проверка:** Если переменная окружения `OPENAI_API_KEY` не установлена, генерация не запускается.

**Логирование:** Пользователю показывается сообщение об ошибке, но это событие НЕ логируется.

**Решение:** Проверить переменные окружения на сервере.

---

### 4. Ошибка в процессе генерации ❌
**Возможные ошибки:**
- Ошибка расчета натальной карты (Swiss Ephemeris)
- Ошибка генерации текста через GPT
- Ошибка создания PDF
- Ошибка отправки PDF пользователю

**Логирование:** Все ошибки логируются как `natal_chart_error` с детальной информацией:
- `error_type` - тип ошибки
- `error_message` - сообщение об ошибке
- `stage` - этап, на котором произошла ошибка (generation, pdf_creation, pdf_send)

---

### 5. Проблема с базой данных 🗄️
**Проверка:** 
- Если используется локальная SQLite БД, нужно проверить локальный файл `users.db`
- Если используется PostgreSQL на Railway, нужно подключиться к удаленной БД

**Решение:** Проверить, какая БД используется (через переменную `DATABASE_URL`).

---

## Рекомендации по диагностике

### Для локальной базы данных:
```bash
python check_all_users.py
```

### Для удаленной базы данных (PostgreSQL):
Нужно подключиться к базе данных Railway и выполнить запросы:
```sql
-- Найти пользователя по username
SELECT user_id, first_name, birth_date, birth_time, birth_place 
FROM users 
WHERE username = 'SberSBI';

-- Проверить события генерации
SELECT event_type, event_data, timestamp 
FROM events 
WHERE user_id = <USER_ID>
AND event_type IN ('natal_chart_generation_start', 'natal_chart_success', 'natal_chart_error')
ORDER BY timestamp DESC;

-- Проверить незавершенные генерации
SELECT e1.timestamp, e1.event_data
FROM events e1
WHERE e1.user_id = <USER_ID>
AND e1.event_type = 'natal_chart_generation_start'
AND NOT EXISTS (
    SELECT 1 
    FROM events e2 
    WHERE e2.user_id = <USER_ID>
    AND e2.event_type IN ('natal_chart_success', 'natal_chart_error')
    AND e2.timestamp > e1.timestamp
)
ORDER BY e1.timestamp DESC;
```
