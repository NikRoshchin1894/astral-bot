# Простая настройка Webhook для Timeweb Cloud 🚀

## Короткий ответ: прокси может быть не нужен!

Если домен уже работает (возвращает 502, а не 404), значит **маршрутизация уже настроена** в Timeweb Cloud!

**502 Bad Gateway** означает:
- ✅ Домен доступен
- ✅ Запросы доходят до контейнера
- ❌ Но Flask сервер не запущен или не отвечает

## Что нужно сделать

### Шаг 1: Установить переменную окружения

В панели Timeweb Cloud добавьте:

**Key:** `TELEGRAM_WEBHOOK_URL`  
**Value:** `https://nikroshchin1894-astral-bot-e77d.twc1.net/webhook/telegram`

### Шаг 2: Перезапустить контейнер

После добавления переменной перезапустите контейнер.

### Шаг 3: Проверить логи

В логах должно появиться:
```
🌐 Запуск бота в режиме WEBHOOK
🌐 Запуск webhook сервера на 0.0.0.0:8080
✅ Webhook сервер запущен
✅ Webhook успешно установлен в Telegram
```

### Шаг 4: Проверить работу

```bash
python diagnose_webhook.py
```

Если теперь получаете **200 OK** вместо **502** - всё работает! ✅

---

## Если всё равно 502

Тогда действительно нужен прокси. В панели Timeweb Cloud:

1. Найдите раздел **"Домены"** или **"Прокси"**
2. Настройте маршрутизацию:
   - `/webhook/telegram` → `localhost:8080`
   - `/webhook/yookassa` → `localhost:8080`

---

## Итог

**Скорее всего прокси НЕ нужен** - просто нужно:
1. Установить `TELEGRAM_WEBHOOK_URL`
2. Перезапустить контейнер
3. Бот сам запустит webhook сервер

**Прокси нужен только если:**
- После перезапуска всё равно 502
- В панели Timeweb Cloud есть раздел для настройки прокси
- Документация Timeweb Cloud требует явной настройки
